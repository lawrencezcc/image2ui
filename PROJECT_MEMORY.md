# Project Memory

这个文件是项目的单一事实源。

## Memory 1: Project Goal

将 UI 参考图的效果 100% 还原复刻为可视化 UI 组件，支持 `dom`、`svg`、`canvas` 等主流的底层技术。

## Memory 2: Program Design Principle

遵守软件设计的 SOLID 原则，优先保证系统具备清晰职责边界、可扩展性、可替换性、低耦合和可维护性。

## Decision Rules

1. 所有技术方案都以项目目标为第一原则，而不是以单个模型、单个框架、单个实现偏好为第一原则。
2. 当前阶段优先把 `dom + svg` 做到最稳、最可验证、最可修复，不以牺牲效果为代价硬切到 `canvas`。
3. `canvas` 是项目必须达成的正式目标，即使当前阶段暂未完全实现，也必须在架构上保留清晰扩展位，且不能影响 `dom + svg` 的效果上限。
4. 任何生成、验证、修复、评估与模型选型，都应优先服务“视觉还原度、可复盘性、可调试性、可扩展性”。
5. 在程序设计上，应尽量遵守 SOLID 原则，避免把特定模型、渲染技术或检测逻辑硬编码到无法扩展的单体实现中。
